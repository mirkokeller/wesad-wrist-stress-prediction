\begin{abstract}
This project predicts stress from wrist physiological signals in the WESAD dataset. Only Empatica E4 wrist signals are used: blood volume pulse (BVP), electrodermal activity (EDA), skin temperature (TEMP), and three-axis acceleration (ACC). Chest signals from the RespiBAN device are excluded. We study two supervised tasks: binary stress versus non-stress classification, and three-class classification with baseline, stress, and amusement. All results use leave-one-subject-out cross-validation, so every test fold contains one unseen subject. The best binary model is a PyTorch MLP trained on GPU. It reaches 0.845 accuracy, 0.799 balanced accuracy, 0.808 macro F1, and 0.886 stress AUC. The three-class task is weaker because amusement is hard to separate.
\end{abstract}

\section{Problem and Data}

The goal is to understand whether wrist signals can help detect stress states. This is useful because wrist devices are easier to wear than chest sensors. However, this project is not a clinical diagnostic system. It is an academic evaluation on a controlled laboratory dataset.

Repository: \url{https://github.com/Krim0k27/wesad-wrist-stress-prediction}

WESAD contains 15 valid subjects. The protocol includes baseline, stress, amusement, and recovery phases. The ground truth is the protocol label assigned by the experiment. This means the label is not a continuous clinical stress measure.

\section{Preprocessing and Features}

Signals are segmented into sliding windows. ACC uses 5 s windows. BVP, EDA, and TEMP use 60 s windows. The shift is 0.25 s. A physiological window is kept only if the full 60 s window has the same protocol label. This reduces transition leakage between phases.

The project uses a custom feature pipeline. ACC features describe axis and magnitude statistics plus peak frequency. BVP features include heart rate, heart rate variability, spectral HRV power, LF/HF ratio, sample entropy, and peak count. EDA features include raw level, range, tonic level, and phasic activity. TEMP uses basic statistics. Inside each LOSO fold, robust scaling, constant-feature removal, and ANOVA feature selection are fitted only on the training subjects. The held-out subject is transformed afterwards.

The most stable selected features include ACC magnitude, BVP heart-rate and HRV features, and EDA level/range features. These features are useful for model interpretation, but they are not proof of causal medical markers.

\section{Models and Validation}

The original WESAD paper used models such as Decision Tree, Random Forest, AdaBoost, LDA, and kNN. We do not use these models. The final comparison uses:

\begin{itemize}
    \item Logistic Regression, as a linear probabilistic baseline;
    \item Linear SVM, as a margin-based baseline with balanced class weights;
    \item Torch MLP, as a neural network trained on CUDA;
    \item Torch MLP Balanced, the same neural network with class weights.
\end{itemize}

Validation uses leave-one-subject-out cross-validation. In each fold, 14 subjects are used for training and one subject is used for testing. This is important because windows from the same subject are highly related. A random window split would overestimate performance.

The main metrics are macro F1 and balanced accuracy, because the classes are imbalanced. Accuracy, precision, recall, confusion matrices, per-subject errors, and ROC/AUC are also reported. ROC/AUC is computed for the probabilistic models.

\section{Critical Analysis}

The results support the binary formulation more than the three-class formulation. Stress and non-stress are easier to separate than baseline, stress, and amusement. The per-subject error plots in Figure~\ref{fig:subject-error} show that performance is not uniform across people. This is a key limitation for healthcare use.

There are several limits. WESAD has only 15 valid subjects, mostly young adults. The labels come from protocol timing, not from clinical diagnosis. Overlapping windows are similar to each other. ACC may capture task context, movement, or posture. BVP and EDA can change because of sensor contact, skin properties, and motion. No external validation is included.

More advanced AI could be useful with larger datasets, longer recordings, and external validation. In this project, a small MLP is enough to test the idea, but it does not make the method ready for clinical use. A real system would need consent, secure storage, data minimization, clear user communication, fairness checks, and monitoring for model drift.

\section{Conclusion}

The project meets the main requirements. It uses only Empatica E4 wrist data, excludes chest signals, extracts physiological features, compares several allowed models, evaluates binary and three-class tasks, and uses LOSO validation. The strongest result is the binary Torch MLP with 0.845 accuracy, 0.799 balanced accuracy, 0.808 macro F1, and 0.886 AUC. The three-class task is weaker because amusement is difficult to classify. The model is useful as an academic experiment, but it is not ready for clinical deployment.
